# Battle-Mini-Demo-with-QA-Functionality-Implemented
A small combat system prototype implemented in C++ using Siv3D.
## 3Dアクションゲーム 実装機能一覧

### 0.概要と基本操作
C++/Siv3Dを使用した戦闘デモゲームです。プレイヤーはキャラクターを操作し、近づいてくる敵を近接・遠距離攻撃を駆使して倒します。
大きな特徴は、デバッグを意識したQAパネルの表示と、当たり判定の可視化やスローモーションなどのデバッグ専用機能の実装です。
アクションゲームにおいて「手触りの良さ」や「理不尽さの排除」は大切ですが、それらを感覚だけで調整するのは困難だと考えました。
そこで、キャラクターの内部ステータスやシステムの状態をリアルタイムで可視化し、バグの早期発見や実務を想定したプランナーとのバランス調整を効率化するために、この専用パネルを実装しました。

キーボード・マウスでの基本的な操作は以下の通りです。
- WSAD：移動
- Space：回避
- 左クリック：近接攻撃
- 右クリック：遠距離攻撃
- TABキー：QAパネルの表示非表示の切り替え
- Hキー：当たり判定の可視化
- Iキー：無敵モード
- Tキー：スローモーション
- E：敵キャラクターのリスポーン

### 1. 基本システムとシーン管理

#### シーン遷移（ステートマシン）

* Title → Play → GameOver / GameClear の状態遷移を実装。

* 死亡時や敵全滅時に適切に結果画面へ移行し、入力待機を経てタイトルへ戻るゲームループを構築。

#### マルチデバイス入力対応

* キーボード＆マウス操作に加え、ゲームパッド（XInput）に対応。

* 左スティックでのアナログな移動入力と、各ボタンへのアクション割り当てを行い、家庭用ゲーム機を想定した操作感を実現。

### 2. プレイヤーアクションと戦闘システム

#### 基本アクション

* 近接攻撃（斬撃）: 自身を中心として、円柱状の攻撃判定を発生。

* 遠距離攻撃（射撃）: 向いている方向へ直進する弾を発射。

* 回避（ドッジ）と i-frame（無敵時間）

入力方向に一定時間高速移動する回避アクションを実装。
発動直後の0.2秒間に i-frame（無敵時間）** を設定することで、敵の攻撃判定と重なってもダメージを無効化してすり抜ける「シビアだがリターンがある」アクションを実現できるよう意識。
また、連打防止のためのクールダウン（0.8秒）を設定。

#### 被弾とダメージ処理

- 敵の攻撃を受けるとHPが減少し、反対方向へのノックバックが発生。

- 被弾後は1.0秒間の無敵時間（キャラクターが点滅）を付与し、連続ダメージを防止。

### 3. エネミー（敵）AI
Chase（接近） → Attack（攻撃） → Recover（硬直） の状態遷移を実装。
単に近づくだけでなく、「射程に入ると攻撃モーションに入り、一定時間後に攻撃判定を発生させ、その後隙ができる」という動きにすることで、
「見てから避けて反撃する」駆け引きが成立できるよう意識

#### 攻撃判定の厳密な制御

- 攻撃モーション中の「0.3秒〜0.5秒」の間のみ攻撃判定が発生するように制御。

### 4. 手触りの向上策

#### ヒットストップ（時間停止）

- プレイヤーの攻撃が敵にヒットした瞬間に0.05秒、プレイヤーが被弾した瞬間に0.1秒、ゲーム内の更新時間を停止（DeltaTimeを0に）する処理を実装。

- ユーザー体験を意識し、攻撃の「重み」や被弾時の「ショック」を視覚・感覚的に強調。

#### ヒットエフェクト（視覚的フィードバック）

- 敵に攻撃が当たった際、一瞬だけ白く発光し、ノックバックする処理を実装。

- 敵の攻撃モーション時（予備動作）には色が濃くなる視覚的な合図（テレグラフ）を実装。

### 5. QA・デバッグ機能（品質保証の取り組み）

実務を想定したスムーズなバランス調整を目的とした専用機能を実装。

#### 当たり判定（Hitbox）の可視化トグル

キー入力で、攻撃判定（赤）、被弾判定（青）、敵の索敵範囲（黄）をリアルタイムに可視化。

#### デバッグ用操作

- スローモーション（処理落ちやフレーム単位での挙動確認用）
- プレイヤーの無敵化（当たり判定や敵のAI挙動のテスト用）
- 敵の任意スポーン（複数体との戦闘テスト用）
- 当たり判定の可視化

#### リアルタイムQAステータスパネル

QAパネル内では、以下を常時表示し、バグの特定を容易に。

- FPS、TimeScale
- Gamepad 接続状況
- プレイヤーの現在のステート（Idle / Dodge / Slash 等）
- i-frame（回避無敵）や被弾無敵の残り秒数
- 最後に受けたダメージの発生源（Last Hit By）とダメージ量
- ヒットストップのタイマー
